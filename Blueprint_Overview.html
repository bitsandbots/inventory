Inventory — Blueprint Overview

Self-hosted Inventory Management System · PHP 8.x + MariaDB 10.x · 97 PHP files · 14 tables · 6 test suites · 62 tests · MIT License

What Is This?

A complete, offline-capable inventory management system that runs on a Raspberry Pi or any LAMP server. Track products with SKUs and images, manage stock levels, process multi-line-item sales orders, print invoices and picklists, and generate sales reports — all through a responsive Bootstrap 5 interface with zero external dependencies.

Inventory Core

  • Products with SKU, location, quantity, pricing
  • Category organization with image support
  • Stock adjustment with comment history
  • AJAX search by name, SKU, or description

Sales & Orders

  • Multi-line-item orders per customer
  • Add sales by SKU or product search
  • Auto quantity decrease on sale
  • Printable invoices and picklists

Reporting

  • Date-range sales reports with profit calc
  • Daily and monthly breakdowns
  • Stock level reports
  • Buy vs. sell price margin analysis

Security & Access

  • 3-tier RBAC (Admin/Supervisor/User)
  • bcrypt passwords + legacy SHA1 migration
  • CSRF protection on all forms + GET deletes
  • CSP, rate limiting, session hardening

Soft-Delete & Trash

  • Reversible delete on users, customers, sales, orders, stock
  • Admin trash UI with restore and purge
  • All list/find queries auto-filter deleted rows
  • Authenticated delete is blocked from soft-deleted accounts

Settings & Configuration

  • DB-backed settings table (key/value)
  • Admin-only settings page (currency code, etc.)
  • Per-install currency with live format preview
  • Configurable via .env for credentials; DB for app settings

Request Lifecycle

1. HTTP Request → index.php or module page
↓ require_once 'includes/load.php'
2. Bootstrap → Constants, session hardening (httponly/samesite/strict_mode), autoload config→functions→session→upload→database→sql→settings
3. Config → Parse .env → DB_HOST, DB_USER, DB_PASS, DB_NAME, APP_SECRET
4. Session → session_start() with strict_mode; Session class initialized
5. Database → MySqli_DB connects, $db global available
6. RBAC Gate → page_require_level(N): login → user active → group active → level ≤ required
7. Handler → POST: verify_csrf() → validate → execute prepared stmt → redirect with flash msg
8. Render → header.php (HTML shell, sidebar by role) → page content with h() escaping → footer.php

Soft-Delete Pattern

Five tables (users, customers, sales, orders, stock) use reversible soft-delete instead of hard DELETE.

Delete → soft_delete_by_id($table, $id) → stamps deleted_at = NOW(), deleted_by = $user
Restore → restore_by_id($table, $id) → clears deleted_at and deleted_by
Purge → purge_by_id($table, $id) → hard DELETE, only allowed when deleted_at IS NOT NULL

find_all() / find_by_id() auto-filter WHERE deleted_at IS NULL (via table_has_soft_delete() probe, cached per request).
find_with_deleted() / find_by_id_with_deleted() bypass the filter for trash UI.

The trash UI is at users/trash.php (Admin-only, level 1). Restore and purge are at users/restore.php and users/purge.php.

Key Abstractions

MySqli_DB (includes/database.php)

// Prepared INSERT/UPDATE/DELETE — returns stmt (check affected_rows)
$db->prepare_query("UPDATE products SET quantity = quantity + ? WHERE id = ?", "ii", $qty, $id);

// Prepared SELECT → array of associative rows
$db->prepare_select("SELECT * FROM products WHERE category_id = ?", "i", $cat_id);

// Prepared SELECT → single associative row or null
$db->prepare_select_one("SELECT * FROM users WHERE username = ? LIMIT 1", "s", $user);

// Escape for table/column name interpolation (never for values)
$db->escape($table_name);

Session (includes/session.php)

Global $session. login($id) calls session_regenerate_id(true) — prevents session fixation. Flash messages (msg()) survive exactly one redirect.

CSRF Protection (includes/functions.php)

echo csrf_field();          // hidden input in every <form method="post">
verify_csrf();              // POST handler entry — returns bool
echo csrf_url_param();      // append to GET-based delete links
verify_get_csrf();          // GET delete handler entry — checks $_GET['csrf_token']

Soft-Delete Helpers (includes/sql.php)

soft_delete_by_id($table, $id)        // stamps deleted_at + deleted_by
restore_by_id($table, $id)            // clears both columns
purge_by_id($table, $id)             // hard DELETE (only when already soft-deleted)
find_by_id_with_deleted($table, $id) // bypasses deleted_at filter (trash UI)
find_with_deleted($table)            // all rows including deleted (trash UI)

Rate-Limiting Helpers (includes/sql.php)

is_login_rate_limited($ip)             // true when >= 5 attempts in 15 min
record_failed_login($ip, $username)    // inserts into failed_logins
clear_failed_logins($ip)              // called on successful login
prune_failed_logins()                 // probabilistic cleanup, no cron needed

SQL Helpers (includes/sql.php)

find_by_id(), find_all() — generic table CRUD with auto soft-delete filter. authenticate() — bcrypt login with legacy SHA1 auto-upgrade. page_require_level(N) — RBAC gate on every protected page.

Authentication & RBAC

LevelRoleAccess
1AdminFull CRUD, user/group management, trash UI, all reports, settings
2SupervisorProduct/stock management, sales, customer CRUD, reports
3UserView products, basic sales entry, order lookup

Login flow: authenticate($user, $pass) → prepared SELECT → IP rate-limit check → password_verify() (or SHA1 legacy) → rehash if needed → session_regenerate_id(true) → updateLastLogIn()

Page guard: page_require_level(N) checks: logged in → user status=1 → group group_status=1 → user_level ≤ N

Quick Start

git clone https://github.com/bitsandbots/inventory.git
cd inventory
bash install.sh

# Access at http://<server>:8080
# Default credentials — change all three immediately:
#   admin / admin   (level 1 — full access)
#   special / special (level 2 — supervisor)
#   user / user     (level 3 — basic)

Reinstall from scratch (wipes database, vhost, symlink, uploads, .env):

bash install.sh --reinstall      # prompts: type RESET to confirm
bash install.sh --reinstall -y   # non-interactive

Full installation steps, Apache vhost config, and troubleshooting: docs/setup-and-usage.md
