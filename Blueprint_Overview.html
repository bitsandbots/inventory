Inventory — Blueprint Overview

Self-hosted Inventory Management System · PHP 8.x + MariaDB 10.x · 79 PHP files · MIT License

What Is This?

A complete, offline-capable inventory management system that runs on a Raspberry Pi or any LAMP server. Track products with SKUs and images, manage stock levels, process multi-line-item sales orders, print invoices and picklists, and generate sales reports — all through a responsive Bootstrap 5 interface with zero external dependencies.

Inventory Core

  • Products with SKU, location, quantity, pricing
  • Category organization with image support
  • Stock adjustment with comment history
  • AJAX search by name, SKU, or description

Sales & Orders

  • Multi-line-item orders per customer
  • Add sales by SKU or product search
  • Auto quantity decrease on sale
  • Printable invoices and picklists

Reporting

  • Date-range sales reports with profit calc
  • Daily and monthly breakdowns
  • Stock level reports
  • Buy vs. sell price margin analysis

Security & Access

  • 3-tier RBAC (Admin/Supervisor/User)
  • bcrypt passwords with legacy SHA1 migration
  • CSRF protection on all forms
  • Parameterized queries throughout

Request Lifecycle

1. HTTP Request → index.php or module page
↓ require_once 'includes/load.php'
2. Bootstrap → Constants, session hardening, autoload config→functions→session→upload→database→sql
3. Config → Parse .env → DB_HOST, DB_USER, DB_PASS, DB_NAME, APP_SECRET
4. Session → session_start() with strict_mode, Session class initialized
5. Database → MySqli_DB connects, $db global available
6. RBAC Gate → page_require_level(N) checks login, status, group, level
7. Handler → POST: verify CSRF → process → redirect. GET: query data → render.
8. Render → header.php (HTML shell, sidebar by role) → page content with h() escaping → footer.php

Key Abstractions

MySqli_DB (includes/database.php)

Database abstraction — global $db instance:

// Prepared INSERT/UPDATE/DELETE
$db->prepare_query("UPDATE products SET quantity = quantity + ? WHERE id = ?", "ii", $qty, $id);

// Prepared SELECT → array of rows
$db->prepare_select("SELECT * FROM products WHERE category_id = ?", "i", $cat_id);

// Single row or null
$db->prepare_select_one("SELECT * FROM users WHERE username = ? LIMIT 1", "s", $user);

Session (includes/session.php)

Login state — global $session instance. login($id) regenerates session ID for fixation protection. Flash messages survive one redirect.

CSRF Protection (includes/functions.php)

Token generated via random_bytes(32), verified with hash_equals() against session-stored token. csrf_field() emits hidden input — use in every <form method="post">.

SQL Helpers (includes/sql.php)

find_by_id(), find_all(), delete_by_id() — generic table CRUD. authenticate() — bcrypt login with legacy SHA1 auto-upgrade. page_require_level(N) — RBAC gate on protected pages.

Authentication & RBAC

LevelRoleAccess
1AdminFull CRUD, user/group management, all reports
2SupervisorProduct/stock management, sales, reports
3UserView products, basic sales entry

Login flow: authenticate($username, $password) → prepared SELECT → password_verify() → session_regenerate_id(true) → updateLastLogIn()

Page guard: page_require_level(N) checks login → user active → group active → level ≤ required

Quick Start

# Clone and run the installer — handles database, app user,
# .env, Apache vhost, permissions in one step.
git clone https://github.com/bitsandbots/inventory.git
cd inventory
bash install.sh

# Then browse: http://<server>:8080
# Login: admin / admin (change immediately!)

To reinstall from scratch (wipes database, vhost, symlink, uploads, .env):

bash install.sh --reinstall      # prompts: type 'RESET' to confirm
bash install.sh --reinstall -y   # non-interactive

Manual install steps and troubleshooting are in docs/setup-and-usage.md.
